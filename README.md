# Весільні онлайн-запрошення (движок на кілька подій)

Один статичний сайт, що під різними URL показує різні запрошення. Кожна подія це тека
`/<slug>/` зі своїм контентом і фото; код і стилі спільні. Секції запрошення: hero з
іменами і датою, зворотний відлік, історія, галерея, таймінг, дрес-код, локації, RSVP,
Telegram, фонова музика. Візуал доводиться у Claude Design і переноситься у `css/`.

## Структура

```
index.html            універсальний рендерер-шаблон (копіюється в кожну подію); / = лендинг
editor.html           візуальний редактор полів + живе превʼю
content.sample.json   шаблон-seed для нової події
css/tokens.css        кольори / шрифти (головний файл вигляду)
css/base.css          база і утиліти
css/sections.css      стилі секцій
js/render.js          збірка DOM з контенту
js/boot.js            вибір контенту за URL/slug
js/*.js               механіки (відлік, галерея, музика, rsvp) + editor.js
apps-script/Code.gs   запасний RSVP-бекенд (Google Apps Script)
design/               бриф і handoff із Claude Design
new-event.ps1         створити нову подію
build-publish.ps1     зібрати publish/ для Netlify

mykhailo-olena/       приклад події:
  index.html          (копія рендерера)
  content.json        контент цієї події
  assets/img, audio   фото і музика цієї події
```

## Кілька подій (один сайт)

- URL події: `/<slug>/`, напр. `site/mykhailo-olena/`. Корінь `/` показує лендинг.
- Додати подію: `./new-event.ps1 <slug>` (див. розділ нижче).
- RSVP усіх подій летить в одну таблицю; події розрізняються по колонках `event` (імена)
  і `page` (URL). Можна дати події окремий SheetMonkey endpoint у її `content.json`.

## Як запустити локально

Сайт використовує `fetch`, тому відкривати через сервер, не подвійним кліком (file://):

```
python -m http.server 5599
```

Далі:
- `http://localhost:5599/studio.html` **Студія** (рекомендовано, Chrome/Edge)
- `http://localhost:5599/<slug>/` подія
- `http://localhost:5599/editor.html` простий редактор (fallback, будь-який браузер)

## Студія (рекомендований редактор, Chrome/Edge)

`studio.html` це локальна адмінка, що працює прямо з папкою проєкту (File System
Access API), один інтерфейс на все:

1. Відкрийте `http://localhost:5599/studio.html`, натисніть **«Відкрити проєкт»** і
   виберіть теку `wedding-invitation`. Дозвольте читання/запис.
2. **Контент:** форма полів, **«Зберегти подію»** пише прямо в `/<slug>/content.json`.
3. **Фото:** кнопка «Замінити» кладе фото в `/<slug>/assets/img/` (з авто-стисненням).
4. **Дизайн:** шрифти і кольори → **«Зберегти дизайн»** пише в `css/tokens.css` (глобально).
5. **Події:** створити/вибрати/видалити подію зі списку зверху.
6. **Публікація:** **«Зібрати publish»** збирає теку `publish/` (з OG) прямо в браузері.

Працює в Chrome/Edge. Це локальний інструмент, у публічну збірку не потрапляє.

## Простий редактор (fallback)

`editor.html` працює в будь-якому браузері: редагує поля і **завантажує** `content.json`,
який ви вручну кладете в теку події. Зручно, якщо немає Chrome.

## Як створити і заповнити подію (без Студії)

1. `./new-event.ps1 <slug>` створює теку `/<slug>/`.
2. `editor.html` → впишіть `<slug>` → **Завантажити подію** → редагуйте → **Завантажити
   content.json** → покладіть у теку `/<slug>/`.

## Фото і музика

- Фото кладіть у теку події `/<slug>/assets/img/` і вказуйте шлях `assets/img/hero.jpg`
  (відносний, відносно теки події). Можна і зовнішній URL.
- Фоновий трек: `/<slug>/assets/audio/background.mp3`. Браузери не дають музиці грати
  самій, тому вона стартує після першого кліку гостя або по кнопці у куті.

## RSVP у Google Sheets

1. Створіть Google Sheet.
2. Extensions -> Apps Script, вставте `apps-script/Code.gs`.
3. Deploy -> New deployment -> Web app: Execute as **Me**, Access **Anyone**. Скопіюйте URL.
4. Вставте URL у редакторі в поле "URL Google Apps Script" (або в `content.json` ->
   `rsvp.appsScriptUrl`).

Відповіді зʼявлятимуться рядками у таблиці. Перевіряти треба саме в таблиці: через
обмеження CORS сайт надсилає дані "не очікуючи відповіді", тому в мережевій вкладці
браузера успіх не видно, а запис у таблицю проходить.

Поки URL порожній, форма показує подяку, а дані пише в консоль (для перевірки).

## Деплой (Netlify)

1. `./build-publish.ps1` збирає теку `publish/` (усі події + спільні css/js, без дев-файлів).
2. Netlify -> ваш сайт -> Deploys -> перетягнути папку `publish/`.
3. URL події: `https://<домен>/<slug>/`. Корінь `/` показує лендинг.

Підходить будь-який статичний хостинг (Netlify, Cloudflare Pages, GitHub Pages).

## Оновлення дизайну через Claude Design

Дизайн навмисно відокремлений від логіки і контенту:

- Згенеруйте/доведіть візуал у Claude Design за `design/claude-design-brief.md`.
- Export -> standalone HTML у `design/handoff/`.
- Скажіть Claude Code інжещнути handoff: оновляться лише `css/tokens.css` і
  `css/sections.css`. Контент (`content.json`), механіки і редактор лишаються.
